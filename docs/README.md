# Documentation Layout

This repository uses a four-layer docs architecture:

- `architecture.md` - top-level system architecture and technology stack.
- `design/` - feature and product design docs (what and why).
- `modules/` - implementation-level module docs (how).
- `knowledge/` - external references, official docs links, and working notes.

## Reading Order

1. `docs/architecture.md`
2. Relevant `docs/design/*.md`
3. Relevant `docs/modules/*.md`
4. Relevant `docs/knowledge/*.md`

## Authoring Rules

- Keep architecture focused on system-level concerns.
- Keep design docs decision-oriented and explicit about tradeoffs.
- Keep module docs code-grounded with concrete entry points and contracts.
- Keep knowledge docs source-linked and version-aware.
