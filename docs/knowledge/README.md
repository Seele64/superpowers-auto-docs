# Knowledge References

Knowledge docs capture external references and reusable notes.

## Use this folder for
- Official documentation links
- RFCs and standards references
- External best-practice summaries
- Project-specific interpretation of external guidance

## Quality bar
- Include source links and versions.
- Record last verification date.
- Explain how the external guidance maps to this project.

## Naming
Use reference-topic filenames, for example:
- `github-actions-cache.md`
- `markdown-style-guidelines.md`
